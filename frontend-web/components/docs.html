Documentação da Plataforma

Referência completa para integrar os microsserviços de KYC e Assinatura Eletrônica em sua aplicação. Todos os endpoints estão documentados em português.

Visão Geral

A Valeris disponibiliza 4 microsserviços plug-and-play de captura de identidade. Cada microsserviço pode ser incorporado de forma independente em qualquer aplicação mediante um token de acesso.

✏️ Assinatura Digital
service_type: "assinatura"
📁 MinIO: assinatura/
📄 Documento
service_type: "documento"
📁 MinIO: documento/
🤳 Selfie Facial
service_type: "selfie"
📁 MinIO: selfie/
🪪 Selfie + Documento
service_type: "selfie-documento"
📁 MinIO: selfie-documento/

Quickstart — Integração em 3 passos

Passo 1 — Defina seu token de acesso

# Modo aberto: sem login/cadastro # Use um token de integração (Bearer) configurado no backend export VALERIS_TOKEN="SEU_TOKEN_DE_ACESSO"

Passo 2 — Incorpore o microsserviço via iFrame

<!-- Coloque este código onde quiser o componente na sua página --> <iframe src="http://API_HOST/?token=SEU_TOKEN_DE_ACESSO&service=assinatura" width="700" height="600" style="border:none;border-radius:16px" allow="camera;microphone" ></iframe>
💡 Troque service=assinatura por service=documento, service=selfie ou service=selfie-documento para usar outro microsserviço.

Passo 3 — Use o SDK JavaScript diretamente

<!-- Importe o SDK --> <script src="http://API_HOST/components/valeris-sdk.js"></script> <script> // Inicializar com seu token ValerisSDK.init({ apiUrl: "http://API_HOST/api", token: "SEU_TOKEN_DE_ACESSO" }); // Enviar uma captura manualmente const geo = await ValerisSDK.getGeolocation(); const dev = await ValerisSDK.getDeviceInfo(); const meta = { ...geo, ...dev, serviceType: "Assinatura Digital" }; const img = await ValerisSDK.generateComposite( [{ label: "Assinatura", dataUrl: minhaImagemBase64 }], meta ); const res = await ValerisSDK.submitCapture("assinatura", img, meta); console.log("Salvo no MinIO:", res.image_data); </script>

Token de Acesso

A API opera em modo token-only (sem login/cadastro). Inclua o header Authorization: Bearer <token> em todas as requisições.

Capturas

Endpoint unificado para todos os microsserviços. O campo service_type determina em qual pasta do MinIO a imagem será salva.

O SDK do Valeris automatiza a coleta de metadados, validação de imagem (nitidez, luz, cor) e geração de composições para segurança máxima.

HTML / JS
<script src="valeris-sdk.js"></script>
<script>
  // Inicialização e captura automática
  const result = await ValerisSDK.validation.face(imgData);
  if(result.isValid) {
      console.log("Confiança:", result.confidence); // 0-100
      // Envio com metadados automáticos
  }
</script>

Parâmetros de Validação

CampoTipoDescrição
confidenceFloatPontuação de 0 a 100 baseada em nitidez, enquadramento e biometria.
detectedTypeStringAuto-identificação: CNH, RG ou Passaporte (Baseado em Aspect Ratio).
isValidBoolConfirma se a imagem atende aos requisitos mínimos de qualidade.

Chaves de API

⚠️ Chaves de API são para integrações server-to-server. Nunca exponha a chave no frontend de produção.
GET/api/keys/
Lista suas chaves de API
POST/api/keys/
Cria nova chave de API
PATCH/api/keys/:id
Atualiza uma chave existente
DELETE/api/keys/:id
Revoga uma chave de API

SDK JavaScript — Referência

O valeris-sdk.js está disponível em /components/valeris-sdk.js e expõe o objeto global ValerisSDK.

ValerisSDK.init(options)

ValerisSDK.init({ apiUrl: "http://sua-api.com/api", // URL base da API token: "SEU_TOKEN_DE_ACESSO" // Token obrigatorio });

ValerisSDK.getDeviceInfo() → Promise

Coleta metadados do dispositivo: IP público, navegador, SO, resolução, fuso horário, data/hora.

ValerisSDK.getGeolocation() → Promise

Coleta coordenadas GPS e resolve o endereço via Nominatim (OpenStreetMap).

ValerisSDK.generateComposite(images, metadata) → Promise<dataUrl>

Gera uma imagem composta com as capturas e um bloco de metadados visível na imagem salva no MinIO.

ValerisSDK.submitCapture(serviceType, compositeDataUrl, metadata) → Promise

Envia a imagem para a API. A API salva no MinIO na pasta correspondente ao serviceType.

Embed via iFrame

A forma mais simples de integrar é via iFrame. Toda a interface e lógica ficam isoladas dentro do iFrame.

<!-- Integração completa: só coloque o token e o service --> <div style="border-radius:20px;overflow:hidden;box-shadow:0 20px 60px rgba(0,0,0,0.4)"> <iframe src="http://API_HOST/?service=assinatura&token=SEU_TOKEN" width="760" height="640" style="border:none" allow="camera;microphone;geolocation" /> </div>

Guia de Integração — Flutter & Go

Abaixo, exemplos de como integrar o Valeris em projetos reais utilizando as tecnologias mais comuns do mercado.

1. Flutter (Injeção de WebView)

Para o Flutter, a melhor forma é utilizar o pacote webview_flutter. Você injeta o token via URL e escuta o evento de sucesso através de postMessage.

// No seu widget Flutter WebViewController controller = WebViewController() ..setJavaScriptMode(JavaScriptMode.unrestricted) ..addJavaScriptChannel('ValerisChannel', onMessageReceived: (message) { if (message.message.contains('VALERIS_CAPTURE_SUCCESS')) { print("Captura realizada com sucesso!"); // Fechar WebView e prosseguir } }) ..loadRequest(Uri.parse("http://VALERIS_HOST/?service=selfie&token=$userJwtToken"));

2. Go (Validação no Backend)

Se você estiver usando Go no backend, pode validar a autenticidade das capturas consultando a API do Valeris ou verificando a assinatura do JWT.

func checkCapture(captureID string, token string) { client := &http.Client{} req, _ := http.NewRequest("GET", "http://VALERIS_API/api/captures/"+captureID, nil) req.Header.Set("Authorization", "Bearer "+token) resp, err := client.Do(req) if err == nil && resp.StatusCode == 200 { fmt.Println("Captura validada com sucesso") } }
🚀 Dica de Produção: Em produçao, use sempre HTTPS para que a câmera funcione corretamente no Chrome e Safari Mobile.
✅ A imagem resultante é automaticamente salva no MinIO com todos os metadados embutidos (IP, localização, data/hora, dispositivo). Nenhuma configuração adicional é necessária no lado do consumidor.
